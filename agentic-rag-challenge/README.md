# Agentic RAG Challenge

This repository is redeveloped around the planning documents in `QUICK_START.md`, `EVALUATION_FRAMEWORK.md`, `CATALOG_SOURCES.md`, `MASTER_SUMMARY.md`, and `ARCHITECTURE_PLAN.md`.

## Target architecture

- Curated Berkeley JSONL records for courses, requirements, and policies
- LangChain-style ingestion, chunking, embeddings, and FAISS retrieval pipeline
- Retrieval-augmented prerequisite and planning runtime with deterministic fallback
- Streamlit demo

## Setup

```bash
cd agentic-rag-challenge
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

The assistant can fall back to local curated records for development, but full assessment-mode RAG is enabled when you build the FAISS index first.

## Run

Build the RAG index:

```bash
python src/build_index.py
```

Run the grounded evaluation:

```bash
python evaluation/eval.py
```

Launch the demo:

```bash
streamlit run demo/streamlit_app.py
```

## Notes

- The processed Berkeley data is already populated under `data/processed/`.
- The curated dataset contains 35 source-backed records: 22 course descriptions, 7 program-requirement records, and 6 policy records.
- The full source inventory is listed in `data/sources.json` with URL, section, access date, and note fields.
- The evaluator uses 25 queries across simple prerequisites, chain reasoning, program requirements, and safe abstention.
- `evaluation/results.json` is generated by the evaluation script.
- `src/orchestration.py` uses FAISS retrieval when index artifacts are present and otherwise falls back to the same curated records directly.
- `src/test_assistant.py` contains a small unit-test sanity suite for the assistant runtime.
- `docs/EXAMPLE_TRANSCRIPTS.md` includes the three example transcript types requested by the assessment.
